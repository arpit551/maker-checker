# Task Brief

Build the requested feature in the repository and keep the implementation production-safe.

## Expectations

- Match the requested behavior exactly.
- Prefer the smallest maintainable change.
- Add or update tests for the main path and edge cases.
- Document what changed and any remaining risks.
- If the request is underspecified, do not invent product requirements. Use the earliest cycle to surface scope gaps and the evidence needed to proceed.
